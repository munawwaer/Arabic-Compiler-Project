🔧 التقنيات المستخدمة وشرح وظيفة كل تقنية

🤖
Android SDK 34
الوظيفة: أحدث مجموعة تطوير أندرويد (Android 14). يوفر دعماً محسّناً لإدارة الأذونات، تحسينات في استهلاك البطارية، وميزات أمان متقدمة.
المميزات: Runtime Permissions المحسّن، Background Execution Limits، Enhanced Notification Channels
☕
Java 17 LTS
الوظيفة: لغة البرمجة الأساسية. Java 17 هو إصدار LTS (دعم طويل المدى) يوفر استقراراً وأداءً محسّناً مع ميزات برمجية حديثة.
الميزات: Pattern Matching، Text Blocks، Records، Sealed Classes
🌐
TCP + UDP Protocols
TCP (المنفذ 10001): لإرسال إشارات التحكم (بدء/قبول/رفض/إنهاء المكالمة). يضمن وصول الرسائل بترتيب صحيح.
UDP (المنفذ 10002): لنقل البيانات الصوتية. سريع جداً، مناسب للصوت المباشر.
🎵
Audio Processing 16kHz
الوظيفة: معالجة صوتية بمعدل 16,000 عينة/ثانية، تنسيق PCM 16-bit أحادي القناة. جودة صوت ممتازة مع استهلاك بيانات معقول.
المواصفات: Sample Rate 16kHz، PCM 16-bit Mono، Latency <100ms

📁 تحليل الملفات - شرح تفصيلي لكل ملف ووظائفه

📱 MainActivity.java

المسؤولية: الشاشة الرئيسية للتطبيق - نقطة البداية التي تحتوي على واجهة المستخدم الأساسية وإدارة الأذونات والتنقل بين الشاشات.
الدوال الرئيسية:
• requestPermissions() - طلب الأذونات اللازمة
• showQr() - عرض QR code للجهاز
• scanQr() - مسح QR للأجهزة الأخرى
• showHelp() - عرض التعليمات

⚙️ CallService.java

المسؤولية: محرك المكالمات الأساسي - يدير جميع عمليات المكالمات في الخلفية كخدمة مستقلة لضمان استمرار العمل.
العمليات الحيوية:
• startListening() - إنشاء خادم لاستقبال الاتصالات
• makeCall() - بدء مكالمة صادرة
• startAudioStreaming() - تفعيل تبادل الصوت
• audioSendingLoop() - حلقة إرسال الصوت
• audioReceivingLoop() - حلقة استقبال الصوت

🌐 SignalingProtocol.java

المسؤولية: تنظيم الرسائل المتبادلة بين الأجهزة وتحديد أنواع الرسائل لضمان التفاهم بين الأجهزة المختلفة.
أنواع الرسائل: CALL_REQUEST، CALL_ACCEPT، CALL_DECLINE، CALL_END، AUDIO_DATA، KEEP_ALIVE، ERROR

🌐 البروتوكولات - كيف تتم المكالمة خطوة بخطوة

1
الاكتشاف والإعداد
الجهاز المستقبل ينشئ QR code يحتوي على عنوان IP، ويبدأ خادم TCP على المنفذ 10001. الجهاز المتصل يمسح QR للحصول على معلومات الاتصال.
2
طلب الاتصال
الجهاز المتصل يرسل طلب TCP مع رسالة CALL_REQUEST. الجهاز المستقبل يظهر إشعار "مكالمة واردة" مع معلومات المتصل.
3
الاستجابة
المستخدم يختار "رد" أو "رفض". في حالة القبول، يتم إرسال CALL_ACCEPT وإنشاء قناة UDP للصوت على المنفذ 10002.
4
تبادل الصوت
كل جهاز ينشئ AudioRecord و AudioTrack. تبدأ حلقتان متوازيتان لإرسال واستقبال الصوت كل 20ms بزمن استجابة أقل من 100ms.
